# WebAF

> A modular Python framework for authorized web application security assessments.

WebAF is an extensible security assessment framework designed to automate reconnaissance, application discovery, passive security analysis, JavaScript endpoint extraction, and reporting for modern web applications.

Unlike single-purpose scanners, WebAF separates crawling, endpoint discovery, analysis, plugins, and reporting into independent components, making it easy to extend with new security modules.

---

## Features

### Asset Discovery

WebAF automatically discovers application assets including:

- HTML pages
- JavaScript files
- Swagger/OpenAPI documentation
- robots.txt
- security.txt

The crawler is scope-aware and configurable through page and depth limits.

---

### JavaScript Endpoint Analysis

WebAF statically analyzes downloaded JavaScript to discover application endpoints before authentication.

Currently supported patterns include:

- fetch()
- Axios
- XMLHttpRequest
- jQuery AJAX
- GraphQL endpoint detection

Extracted endpoints are normalized to produce a clean inventory.

Example:

```
POST /{param}/rest/basket/{param}/checkout
GET  /{param}/rest/products/search
PUT  /{param}/rest/basket/{param}/coupon/{param}
```

---

### Browser Runtime Discovery

(Optional)

When Playwright is installed, WebAF can launch a headless Chromium browser to observe runtime HTTP requests generated by modern JavaScript applications.

Runtime discovery complements static analysis by capturing:

- fetch()
- XMLHttpRequest
- dynamically generated API requests

---

### Endpoint Inventory

Every discovered endpoint is normalized and deduplicated before being passed to plugins.

Each inventory entry includes:

- HTTP method
- Endpoint path
- Content-Type (when available)
- Source JavaScript file

---

### Modular Plugin Architecture

Security checks are implemented as independent plugins.

Current modules include:

| Module | Description |
|----------|-------------|
| Headers | Security header analysis |
| Cookies | Cookie security checks |
| CORS | Cross-Origin Resource Sharing analysis |
| HTTP Methods | Dangerous HTTP method detection |
| Swagger/OpenAPI | API documentation discovery |
| robots.txt | robots.txt analysis |
| security.txt | RFC 9116 validation |
| XXE | XML endpoint identification |

Adding new plugins requires no changes to the framework core.

---

### Reporting

WebAF generates:

- Rich terminal output
- JSON inventory
- JSON plugin results
- HTML assessment report

Reports are written to:

```
reports/
```

# Installation

Clone the repository:

```bash
git clone https://github.com/cyberhacky/webaf.git
cd webaf
```

Create a virtual environment:

```bash
python3 -m venv .venv
source .venv/bin/activate
```

Install dependencies:

```bash
pip install -r requirements.txt
```

Optional browser support:

```bash
python -m playwright install chromium
```

---

# Usage

Basic assessment:

```bash
python3 webaf.py https://example.com
```

Passive assessment with browser discovery:

```bash
python3 webaf.py \
    https://example.com \
    --browser \
    --rate-limit 1 \
    --max-pages 25 \
    --max-depth 2
```

Authenticated assessment using a bearer token:

```bash
export API_TOKEN=<token>

python3 webaf.py \
    https://example.com \
    --bearer-token-env API_TOKEN
```

---

# Command Line Options

| Option | Description |
|---------|-------------|
| `--active` | Enable state-changing requests |
| `--browser` | Enable Playwright runtime discovery |
| `--rate-limit` | Maximum requests per second |
| `--timeout` | HTTP timeout |
| `--retries` | Retry temporary failures |
| `--max-pages` | Maximum crawl pages |
| `--max-depth` | Maximum crawl depth |
| `--allow-host` | Additional in-scope host |
| `--insecure` | Disable TLS verification |
| `--headers-file` | Additional request headers |
| `--cookies-file` | Session cookies |
| `--bearer-token-env` | Bearer token environment variable |
| `--include-inferred` | Include inferred API routes |
